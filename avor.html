Story

Why AVOR?

A disagreement with a car's infotainment system, built into an answer.
AVOR is a fictional premium marque — not a product, a position: design can be quiet.

This project is position-driven, not research-driven. It began with a specific frustration with an existing product, rather than a research synthesis. Decisions are grounded in competitive analysis and direct experience. The artifacts are intentionally scoped and actively being built across web, app, and infotainment.

Philosophy

Silent Presence.

Grounded in a narrative of functional purity — nothing excessive, nothing superfluous.
Every decision earned its place by saying less.

The brand manual was the longest single piece of work in the project. It was written first and constrained everything that followed. Every decision is derivable from a rule in this document. What is not defined is not interpreted.

Infotainment

Simplicity.

The answer to driving interfaces that prioritize feature count over clarity.
Physical controls. Digital precision. Nothing in between.

Built around physical-first interaction — rotary controller and fixed navigation buttons — with touch as optional parallel input. The intent is muscle memory over visual demand. On-screen elements do not initiate; they reflect physical input. The prototype is keyboard-driven because the keyboard maps most directly to the physical control model — arrows for the wheel, enter for the press, escape for the back button, and tab for sub menus.

App

Functionality.

A companion app designed as an extension, not a separate product.
State without overhead.

The companion app began as a redesign exercise using Audi's app as a reference point, then evolved into AVOR's own interface system. Bottom bar omitted deliberately — the app's purpose is quick overview and remote actions; everything else is secondary. The interface distinguishes between actions and states. Actions are button-shaped. States are flat containers. Shape is semantic, not decorative. The system stays silent by default, reveals itself when needed, and escalates only when ignored.

Website

Clarity.

Built on the same discipline as the brand it carries.
AVOR's attitude, native to web.

Designed mobile-first, then scaled by expanding spacing and layout rather than introducing new elements. Hero prioritizes brand tone over product specs. Model differentiation is carried by environment, not copy. Five principles hold across pages — imagery-driven hierarchy, minimal copy, environment as positioning, mobile-first stacking, grid-based spacing.

Reflection

Closing.

The finished work made the next iteration clearer: document design reasoning in more detail, formalize recurring interface patterns earlier, and structure implementation before scale becomes friction.
